Selection Sort Algorithm

Selection Sort is an in-place comparison sorting algorithm. It divides the input list into two parts: the sublist of items already sorted and the sublist of items remaining to be sorted. Initially, the sorted sublist is empty, while the unsorted sublist contains all elements. The algorithm repeatedly finds the minimum element from the unsorted sublist and moves it to the end of the sorted sublist. This process continues until the unsorted sublist becomes empty.

History

Selection Sort has been known since the 1950s and was likely the method used by the card sorters used in early IBM data processing systems. While not as efficient as more complex algorithms like quicksort or mergesort, it is still useful for sorting small datasets or as a building block for more advanced algorithms.

Working Principle

Selection Sort maintains two subarrays in a given array: one that is already sorted and another that is yet to be sorted. The algorithm repeatedly finds the minimum element from the unsorted portion and moves it to the beginning of the sorted portion. The time complexity of Selection Sort is \(O(n^2)\), making it inefficient for large lists. However, it performs well for small lists due to its simplicity.

Consider an array to be sorted: [5, 3, 8, 4, 2].

Step.1
  • [5, 3, 8, 4, 2] (Initial array)
  • Find the minimum element (2) and swap it with the first element (5)
  • Result: [2, 3, 8, 4, 5]
Step.2
  • [2, 3, 8, 4, 5]
  • Find the minimum element (3) from the remaining array (starting from index 2) and swap it with the second element (3)
  • Result: [2, 3, 8, 4, 5] (No swap needed)
Step.3
  • [2, 3, 8, 4, 5]
  • Find the minimum element (4) from the remaining array (starting from index 3) and swap it with the third element (8)
  • Result: [2, 3, 4, 8, 5]
Step.4
  • [2, 3, 4, 8, 5]
  • Find the minimum element (5) from the remaining array (starting from index 4) and swap it with the fourth element (8)
  • Result: [2, 3, 4, 5, 8]
Result

[2, 3, 4, 5, 8] (sorted)

Time Complexity

  • Best-case time complexity: \(O(n^2)\)
  • Average-case time complexity: \(O(n^2)\)
  • Worst-case time complexity: \(O(n^2)\)

Summary

Selection Sort is a straightforward sorting algorithm that is easy to implement and understand. However, its time complexity makes it inefficient for large datasets, and it is generally not used in practice for sorting large arrays. Nevertheless, it serves as a good educational tool for teaching the concept of sorting algorithms and can be useful for sorting small arrays or as a subroutine in more complex algorithms.

Java Code

        public void sort() {
        int[] list = getList();
        int least;

        for (int i = 0; i < list.length - 1; i++) {
            least = i;

            for (int j = i + 1; j < list.length; j++) {
                if (list[j] < list[least])
                    least = j;
            }
            if (least != i)
                swap(i, least);
        }
        }
